Introduction

Have you ever heard of SQL? I would probably say yes to a student majoring in computer science. But what is up for the others? If your answer is no, do not worry; this article will provide you with some information about SQL. Let's jump right in.

What is SQL?

Structured query language (SQL) is a computer language used to store and manipulate data in relational databases. A relational database is a relational database that stores information in structured format, with rows and columns reflecting distinct data qualities and the various connections between the data values. SQL commands can be used to save, update, delete, search, and retrieve data from a database. SQL may also be used to maintain and improve database performance.

Why SQL is important?

Structured query language (SQL) is a famous query language used in a broad range of applications. SQL is learned and used by data analysts and developers since it interacts effectively with so many programming languages. For example, they can use the programming language Java to integrate SQL queries in order to develop high-performance data processing applications with major SQL database systems such as Oracle or MS SQL Server. SQL is also relatively simple to learn since the statements use familiar English vocabulary.

When to use SQL?

SQL helps in the management of information maintained in databases, giving users to access the specific information they require when necessary. SQL is a powerful programming language despite its straightforwardness. SQL is capable of inserting data into database tables, transforming the data in established database tables, and deleting data from SQL database tables. SQL may also change the database structure by adding, updating, and removing tables and other database objects. SQL is a series of commands that is used to alter data in databases. SQL INSERT is utilized to add data to database tables; SQL SELECT is utilized to retrieve data from database tables; and SQL UPDATE is used to change existing database records.

SQL Syntax

Syntax is the group of rules that govern how language parts are integrated accurately. SQL syntax is based on English syntax and shares many similarities with Visual Basic for Applications (VBA) syntax.

Every SQL statements begin with one of the following terms SELECT, INSERT, UPDATE, DELETE, ALTER, DROP, CREATE, USE, SHOW, and finish with a semicolon (;).

The most essential thing to remember here is that SQL is case insensitive, which means that in SQL statements, SELECT and select have the same meaning. MySQL, on the other hand, distinguishes between table names. So, if you're dealing with MySQL, you'll need to offer table names exactly as they appear in the database.

SELECT Query

The SQL SELECT command retrieves data from a database table and returns it in the form of a result table. These result tables are referred to as result-sets.

SELECT Syntax

SELECT column1,column2, ... FROM table_name;

If you would like to retrieve the entire fields in the field, use the following syntax.

SELECT * FROM table_name;

SELECT * Example

The following SQL statement will retrieve all of the columns of data from employees table

SELECT * FROM employees;

SELECT Column Example

The following SQL statement will retrieve the columns of data (country_id, country_name, and region id) from table countries.

SELECT country_id, country_name, region_id FROM countries;

WHERE Clause

The SQL WHERE clause specifies a criteria for retrieving data from a single table or connecting several tables. Only if the provided condition is met does it return a specific value from the table. You should utilize the WHERE clause to filter the records and get just those that are required.

The WHERE clause is used not only in the SELECT statement, but also in the UPDATE, DELETE, and other statements that we will look at in the following chapters.

WHERE Syntax

SELECT column1, column2, FROM table_name WHERE [condition];

WHERE Clause Example

The following SQL statement will retrieve columns of data title and year from d_cds table where the year is before 2000.

SELECT title, year FROM d_cds WHERE year < 2000;

AND & OR Clauses

The SQL AND and OR operators are used to combine multiple conditions in a SQL statement to narrow data. The conjunctive operators are these two operators.

These operators allow you to perform multiple comparisons with different operators within the same SQL statement

The AND operator Syntax

SELECT column1, column2, columnN FROM table_name WHERE [condition1] AND [condition2]...AND [conditionN];

AND Operator Example

SELECT first_name, last_name FROM f_staffs WHERE salary > 6.50 AND staff_type != 'Order Taker';

Using the AND operator, you may combine N criteria. All conditions separated by the AND must be TRUE for the SQL statement to take action, whether it is a transaction or a query.

The OR Operator Syntax

SELECT column1, column2, columnN FROM table_name WHERE [condition1] OR [condition2]...OR [conditionN];

Using the OR operator, you may combine N conditions. For the SQL statement to perform an action, whether it is a transaction or a query, just ONE of the requirements separated by the OR must be TRUE.

OR Operator Example

SELECT department_name, manager_id, location_id FROM departments WHERE location_id = 2500 OR manager_id = 124;

UPDATE Query

The SQL UPDATE Query is used to alter existing table records. To update just the specified rows, use the WHERE clause with the UPDATE query; otherwise, all rows will be changed.

UPDATE Syntax

UPDATE table_name SET column1 = value1, column2 = value2...., columnN = valueN WHERE [condition];

Using the AND and OR operators, you might well aggregate N requirements.

UPDATE query Example

UPDATE customers SET address = 'Manhattan Street' WHERE salary > 100000;

DELETE Query

To remove existing records from a table, use the SQL DELETE Query.

You may use the WHERE clause in conjunction with a DELETE query to delete only the chosen rows; alternatively, all records are deleted.

DELETE Syntax

DELETE table_name WHERE [condition];

DELETE Query Example

DELETE customers WHERE id = 100;
